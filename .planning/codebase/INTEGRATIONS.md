# External Integrations

**Analysis Date:** 2026-06-21

## APIs & External Services

**Email:**
- Resend - Intended transactional email provider for verification and password reset flows.
  - SDK/Client: Not currently installed or called directly.
  - Auth: `RESEND_API_KEY` read via `server/src/config/configuration.ts`.
  - Current behavior: `server/src/modules/mail/mail.service.ts` logs email content when no key is configured; real dispatch is marked as future wiring.

**External APIs:**
- None currently observed for business features beyond storage/email infrastructure.

## Data Storage

**Databases:**
- PostgreSQL - Primary relational store.
  - Connection: `DATABASE_URL` env var.
  - Client: Prisma via `server/src/infra/prisma/prisma.service.ts`.
  - Schema: `server/prisma/schema.prisma`.
  - Migrations: `server/prisma/migrations/`.
  - Dev container: `postgres:16-alpine` in `docker-compose.yml`.

**File Storage:**
- Cloudflare R2 - S3-compatible storage for uploaded documents/media.
  - SDK/Client: `@aws-sdk/client-s3` in `server/src/infra/storage/storage.service.ts`.
  - Auth env vars: `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`.
  - Bucket env var: `R2_BUCKET`.
  - Behavior: service warns at boot when credentials are incomplete; upload/download throw if not configured.

**Caching / Queues:**
- Redis - BullMQ connection configured in `server/src/app.module.ts`.
  - Connection env vars: `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD`.
  - Dev container: `redis:7-alpine` in `docker-compose.yml`.
  - Used by: `NotificationModule` queue constants/processor scaffold.

## Authentication & Identity

**Auth Provider:**
- Custom email/password auth with JWT access and refresh tokens.
  - Implementation: `server/src/modules/auth/auth.service.ts` and `server/src/modules/auth/strategies/jwt.strategy.ts`.
  - Password hashing: `PasswordService` using argon2.
  - Refresh token storage: hashed JWT refresh tokens in `RefreshToken` table.
  - Token storage on client: persisted Pinia auth store in `client/app/stores/auth.ts`.

**OAuth Integrations:**
- None currently implemented.
