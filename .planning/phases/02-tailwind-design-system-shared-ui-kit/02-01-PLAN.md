<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Page/composable to Ui components | Feature data is untrusted from the UI kit perspective; components must render slots/props without data fetching. |
| Keyboard user to interactive layout | Focus order and visible focus must remain usable after layout primitive adoption. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `client/app/components/ui/*` | mitigate | Keep all `Ui*` components data-agnostic; no API, store, auth, tenant, or feature composable calls. |
| T-02-02 | Denial of Service | responsive layout primitives | mitigate | Use wrapping toolbar/container classes and `min-w-0` patterns to prevent overflow and clipped controls. |
| T-02-SC | Tampering | npm package surface | mitigate | No package install in this plan; if an install appears, stop and require Package Legitimacy Audit update. |
</threat_model>
