Purpose: Satisfy UIKIT-04 using accessible, semantic, Tailwind-only primitives shaped by current toast, skeleton, dialog, tab, alert, and empty-state needs.
Output: `UiAlert`, `UiToast`, `UiSkeleton`, `UiEmptyState`, `UiProgress`, `UiDialog`, `UiConfirmDialog`, and `UiTabs`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller workflow to dialog primitives | Dialogs receive open state and emit events; feature save/delete remains caller-owned. |
| Toast/alert content to assistive tech | Feedback needs semantic roles and must not hide critical errors. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Denial of Service | `UiDialog`, `UiConfirmDialog` | mitigate | Implement Escape/backdrop/focus behavior and visible close/cancel controls. |
| T-02-12 | Tampering | `UiToast`, `UiAlert`, `UiEmptyState` | mitigate | Avoid `v-html`; render escaped text/slots and caller-owned copy. |
| T-02-13 | Repudiation | destructive confirmation | mitigate | Require explicit caller labels/messages and explicit `confirm`/`cancel` events. |
| T-02-SC | Tampering | npm package surface | mitigate | No package install in this plan. |
</threat_model>
