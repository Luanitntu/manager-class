# Plan 04-06: Close Visual Parity UAT Gaps

## Goal

Bring the Phase 4 priority pages into visual alignment with the newer teacher Calendar and Dashboard surfaces while preserving existing data flows and behavior.

## UAT Gaps

- `/assistants` has the right pieces but does not feel polished or consistent with Calendar.
- `/audit-logs` has the right pieces but still feels like a plain table/admin page, not a newer Schedule Teacher surface.
- `/profile` is too sparse as a standalone page; the primary profile flow should be modal-first from the shell profile menu.
- The four redesigned pages are internally consistent but not sufficiently aligned with newer codebase UI.

## Constraints

- Do not change backend APIs.
- Keep feature composables and route-owned data flow.
- Preserve pagination, filters, create/detail behavior, salary/profile save behavior, loading/empty/error/success states.
- Use Tailwind and existing `Ui*` primitives only.
- Do not reintroduce Vuetify, SCSS, new dependencies, or direct API calls inside shared UI components.

## Tasks

### 1. Establish Calendar/Dashboard Visual Parity Tokens

Reference:

- `client/app/components/calendar/TeacherCalendar.vue`
- `client/app/components/calendar/TeacherCalendarHeader.vue`
- `client/app/components/dashboard/TeacherWorkspaceDashboard.vue`
- `client/app/layouts/default.vue`

Extract the concrete visual pattern to reuse:

- shell background and white bordered operational surfaces
- compact `UiPage` rhythm, `gap-6`, `rounded-[var(--st-radius)]`, `border-[var(--st-border)]`
- icon-led section headers
- semantic metric/list surfaces
- primary blue plus restrained orange/green/violet accents
- dense but polished rows with strong leading metadata

### 2. Restyle `/assistants`

Modify `client/app/pages/assistants/index.vue`.

Expected result:

- Header/action area feels like Calendar/Dashboard, not a generic CRUD list.
- Search and create action sit in a polished operational toolbar.
- Assistant rows use stronger visual hierarchy, icon/avatar treatment, and badge rhythm matching dashboard list rows.
- Empty/loading/create dialog states remain present and behavior-compatible.
- No mobile overflow, overlap, clipped controls, or unreadable badges.

### 3. Restyle `/audit-logs`

Modify `client/app/pages/audit-logs.vue`.

Expected result:

- Filter toolbar and table surface look like a first-class Schedule Teacher operational view.
- Add icon-led section treatment and better tone hierarchy without adding new data or actions.
- Keep table columns, filters, clear filters, badge mapping, loading skeleton, empty state, and pagination.
- Mobile only scrolls inside the table container.

### 4. Convert Profile Primary Flow To Modal-First

Modify:

- `client/app/layouts/default.vue`
- `client/app/pages/profile.vue`

Expected result:

- Shell profile menu opens a polished profile dialog for the primary view/edit flow.
- Dialog preserves `useMyProfile`, `useProfileMutations`, timezone list/fallback, save loading, inline error, inline success, auth store updates, and avatar key update.
- `/profile` remains available as fallback or becomes a richer account page that does not feel sparse.
- No duplicated API ownership in shared UI components.

### 5. Cross-Page Parity Sweep

Check all four routes:

- `/assistants`
- `/assistants/[id]`
- `/audit-logs`
- `/profile`

Expected result:

- They align with Calendar/Dashboard, not just with each other.
- No horizontal page overflow except intentional table scroll.
- No overlap, clipped controls, unreadable text, or broken wrapping.
- Loading, empty, error, success, disabled, and dialog states remain reachable.
